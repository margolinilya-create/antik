# Антик — антикварный интернет-магазин

Курируемая витрина антиквариата для российского рынка: подбор и продажа
уникальных предметов в наличии (самовары, иконы, фарфор, мебель, монеты,
живопись). Каждый предмет — в единственном экземпляре.

## Стек

Next.js 16 (App Router, TypeScript) · React 19 · Supabase (Postgres/Auth/Storage)
· Tailwind CSS 4 · Vercel.

## Что уже есть (фундамент)

- **Каталог + SEO**: листинг, карточка предмета с `Product`+`Offer` JSON-LD,
  хлебные крошки, `sitemap.xml`, `robots.txt`, ISR + on-demand ревалидация.
- **Поиск и подбор**: фасетные фильтры (категория, эпоха, мастер, материал,
  цена) + русский полнотекстовый поиск (Postgres `tsvector` + `pg_trgm`) через
  единую RPC `search_items`.
- **Заявки**: форма «оставить заявку / забронировать» → таблица `inquiries`
  (RLS: анонимный insert), уведомление владельцу в Telegram.
- **Безопасность**: RLS на всех таблицах, публичное чтение только опубликованных
  предметов, запись только для админа.

## Быстрый старт

```bash
pnpm install
cp .env.example .env   # заполнить ключами Supabase (опционально для UI)
pnpm dev               # http://localhost:3000
```

Приложение запускается и без Supabase — каталог покажет пустое состояние.

## Подключение Supabase

1. Создайте проект на [supabase.com](https://supabase.com) и пропишите ключи в `.env`.
2. Примените миграции и сиды (SQL Editor или CLI):
   `supabase/migrations/0001_init.sql` → `0002_search.sql` → `supabase/seed.sql`.
3. Создайте публичный bucket `item-images` в Storage.
4. Заведите админа: зарегистрируйте пользователя в Auth и добавьте строку в
   `profiles` с `role = 'owner'`.
